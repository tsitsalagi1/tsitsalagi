Ꮳ TsitsalagiSetup guide

Admin setup

Make the site easy to update

Use Google Forms for submissions, Google Sheets for moderation, and public CSV links for approved listings, issues, and resources.

1. Create three Google Sheets

You can also use one spreadsheet with three approved tabs. The important part is that only approved rows are published.

Approved Listings columns

Approved, Title, Category, Area, Price, Description, Contact, Posted, Expires, Tags

Approved Issues columns

Approved, Title, Category, Status, Area, Question, Ask, Source, LastUpdated

Resources columns

Approved, Title, Category, Description, Link

2. Publish the approved tabs as CSV

  1. Open the Google Sheet.
  2. File → Share → Publish to web.
  3. Choose the approved tab only.
  4. Choose CSV / comma-separated values.
  5. Copy the published CSV link.

Do not publish the raw form response tab unless you are okay with unreviewed submissions being public.

3. Paste links into config.js

window.TSITSALAGI_CONFIG = {
  listingsCsvUrl: "https://docs.google.com/spreadsheets/d/e/.../pub?output=csv",
  issuesCsvUrl: "https://docs.google.com/spreadsheets/d/e/.../pub?output=csv",
  resourcesCsvUrl: "https://docs.google.com/spreadsheets/d/e/.../pub?output=csv",
  listingFormUrl: "https://forms.gle/...",
  issueFormUrl: "https://forms.gle/...",
  contactEmail: "your-email@example.com"
};

After this, daily updates happen in Google Sheets. GitHub only needs changes when you redesign the site.

4. Moderation workflow

  1. Citizen submits a form.
  2. You review the private response.
  3. Copy approved rows to the approved public tab.
  4. Delete or ignore unsafe/incomplete submissions.
  5. The public site updates automatically after Google refreshes the CSV.

5. Start narrow

Launch with a few categories first: Free/Giveaway, Buy/Sell/Trade, Services, Jobs/Side Work, Events, and Citizen Issues. Add more only after moderation is manageable.
